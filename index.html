StrokeDB is an embeddable distributed document database written in Ruby. It is schema-free, it scales infinitely, it even tracks revisions and perfectly integrates with Ruby applications.

Motivation

Relational databases are widespread and well-known. However, in RDBMS we constantly meet hard to solve issues: schema migrations, large table schema updates, continuous data sharding, partial replication.
We are trying to solve these issues with StrokeDB.

Check out this article on dumb databases and read/write consistency (September 20, 2007). It describes the main issues with RDBMS and clarifies our intentions. Actually, I’ve never seen it before today (March 24, 2008), but it is so-o true. It is a must-read for everyone interested in a StrokeDB.

Features

  • Flat address space of documents identified by UUIDs.
  • JSON, schemaless document format.
  • Flexible object-oriented API for documents operations. Much nicer and consistent than ActiveRecord API.
  • Meta documents with serialized definitions and executable code mixins.
  • Documents revision control with diff/merge facilities built-in.
  • References to other documents with automatic eager loading on access.
  • Chunked storages (allowing you to distribute the data across the cluster).
  • Storage chains facility for load balancing.
  • Simple search index over document slots.
  • UTF-8 support built-in. No need to configure encoding in 23 different places like in M*SQL.
  • Two major Ruby implementations support (Ruby 1.8.6, JRuby 1.1RC1).
  • Unit-, integration- and regression tests using RSpec.

Current status

StrokeDB is an experiment. It is only 2.5 months old and contains a clean and working code along with crazy experiments and API inconsistencies.

For now, you can use it as a replacement for SQLite, however it doesn’t support complex queries. Current version includes nice object-oriented API, revision control, simple general-purpose index, memory- and file-based storages, simple DRb interface, IRb console and incremental views (like in CouchDB).

Also, there are some useful facilities interesting for developers only. Like a collection of diff/merge algorithms or a portable class optimization framework (allowing you to optimize ruby methods with a native code).

You may track progress using Gitorious, our primary repository hosting.

Roadmap

The first “serious” release of the StrokeDB will include all the features declared above.

Further plans include (higher position — higher priority):

  • Flexible API for custom search indexes.
  • Advanced network protocols with EventMachine and EM Utils for distributed access and realtime load analysis.
  • Batch data import from ActiveRecord.
  • Simple HTTP browser and editor (something like git-web, phpMyAdmin etc.).
  • Advanced associations and validations API.
  • Various optimizations using native code (C, Java) and smarter algorithms to preserve data locality and save network bandwidth.
  • Support for Ruby 1.9 and Rubinius. Ruby 1.9 is a real thing already, however major code review is neccessary. Rubinius doesn’t support important libraries yet (rubygems, activesupport).
